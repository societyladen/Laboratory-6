### Алгоритм решения



Программа начинает работу с вывода приглашения ввести данные для создания питомца. Для ввода клички используется метод CheckString() из класса Checker, который запускает цикл: читает строку с консоли и проверяет, что она не пустая. Если строка пустая, выводится сообщение об ошибке и ввод повторяется. Для ввода возраста используется метод CheckInt(min, max) из класса Checker, который в бесконечном цикле читает строку, пытается преобразовать её в целое число с помощью int.TryParse(), и если преобразование успешно и число попадает в диапазон от 0 до 50, возвращает число; иначе выводит сообщение об ошибке и повторяет ввод. Для ввода логических значений (хищник, охотится ночью) используется метод CheckBool(), который проверяет, что пользователь ввёл "true"/"false"/"t"/"f"/"1"/"0". После успешного ввода всех данных создаётся объект namedPet1 класса NamedPet (дочерний класс) через конструктор с параметрами, который вызывает конструктор базового класса Pet через ключевое слово base() для инициализации полей IsPredator и HuntAtNight, а затем сохраняет имя и возраст в собственные поля. Затем демонстрируется работа конструктора копирования. Создаётся объект namedPet2 как копия namedPet1. Конструктор NamedPet(NamedPet other) вызывает конструктор копирования базового класса Pet(Pet other), который копирует поля IsPredator и HuntAtNight, после чего копирует поля name и age. После этого программа демонстрирует метод Age(), который увеличивает возраст питомца на заданное количество лет (предварительно проверяя, что количество лет положительное). Выводится текущий возраст, пользователь вводит количество лет, и возраст обновляется. Затем демонстрируется метод IsDangerousForChild(), который вызывает метод базового класса Disjunction() (вычисляющий дизъюнкцию полей IsPredator и HuntAtNight). Если дизъюнкция возвращает true, метод возвращает предупреждение об опасности; иначе — сообщение о безопасности. Далее демонстрируется второй конструктор класса NamedPet (только имя и возраст). Пользователь вводит кличку и возраст, создаётся объект namedPet3, при этом поля IsPredator и HuntAtNight получают значения по умолчанию (false и false). Объект выводится на экран через переопределённый метод ToString(), который сначала вызывает base.ToString() для вывода полей родителя, а затем добавляет имя и возраст. Наконец, для базового класса Pet демонстрируются: конструктор с параметрами (создание pet1), конструктор копирования (создание pet2 как копии pet1), конструктор без параметров (создание pet3 со значениями false), метод Disjunction() (вычисляющий логическое ИЛИ полей) и метод ToString().

### Алгоритм решения



Программа начинает работу с вывода приглашения ввести координаты двух отрезков. Для ввода координат используется метод CheckDouble() из класса Checker, который в бесконечном цикле читает строку с консоли, пытается преобразовать её в вещественное число с помощью double.TryParse(). Если преобразование успешно, число возвращается; если нет — выводится сообщение об ошибке и ввод повторяется. После ввода всех четырёх координат (x1, y1, x2, y2) создаются объекты s1 и s2 класса LineSegment через конструктор с параметрами. Затем программа демонстрирует унарную операцию ! (восклицательный знак), которая перегружена для вычисления длины отрезка. Внутри оператора ! вычисляется абсолютная разность между координатами y и x. Результат выводится на экран. Далее демонстрируется унарная операция ++ (инкремент), которая перегружена для расширения отрезка на 1 влево и вправо. Оператор создаёт новый объект LineSegment, уменьшая координату x на 1 и увеличивая координату y на 1. Исходный отрезок не изменяется, новый отрезок выводится на экран. После этого демонстрируются операции приведения типов. Неявное приведение к int (оператор implicit) возвращает целую часть координаты x. Присваивание int x = s1; автоматически вызывает этот оператор. Явное приведение к double (оператор explicit) возвращает координату y. Для его вызова используется синтаксис (double)s1. Затем демонстрируются бинарные операции. Оператор - (вычитание) перегружен в двух вариантах: LineSegment - int уменьшает координату x на заданное число, а int - LineSegment уменьшает координату y на заданное число. Оба варианта возвращают новый объект. Оператор < перегружен для проверки пересечения отрезков. Внутри оператора определяются левые и правые концы каждого отрезка (с помощью Math.Min() и Math.Max()). Отрезки пересекаются, если не выполняется условие a2 < b1 || b2 < a1 (то есть если один отрезок не находится целиком левее другого). Оператор возвращает true при пересечении, false в противном случае. Наконец, демонстрируется конструктор копирования. Создаётся объект copy как копия s1. Конструктор LineSegment(LineSegment other) копирует значения полей x и y. Копия выводится на экран.
